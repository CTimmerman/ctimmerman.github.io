--- layout: post title: How to manually mount a USB storage device in GNU/Linux date: '2014-08-26T18:20:00.000+02:00' author: Cees Timmerman tags: - GNU/Linux - terminal - USB - code - mount - bash - shell modified_time: '2014-10-16T10:03:17.635+02:00' blogger_id: tag:blogger.com,1999:blog-6265571233418601933.post-1972453823561103095 blogger_orig_url: https://ceestimmerman.blogspot.com/2014/08/how-to-manually-mount-usb-storage.html --- After creating a bootable (possibly smaller than 16 GB for BIOS compatibility) USB stick, using Universal USB Installer or equivalent, you may find that some distros, like System Rescue CD do not automount your other/larger USB storage devices.

To solve that, run these commands from your terminal: 

mkdir stick
fdisk -l
mount /dev/sdc1 stick
That will:

1. create a mountpoint
2. allow you to find the device that matches your storage size, e.g. /dev/sdc
3. mount the 1st partition of that device on your mountpoint

Before removing the device you may want to unmount it, lest OSs like Windows complain on insert: 
umount stick
You can possibly save keystrokes by pressing the up arrow key a few times so you only have to prepend a "u" to the mount command in your command history: 
umount /dev/sdc1 stick
